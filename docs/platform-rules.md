# 小红书 RED Skill 平台规则

> 来源：官方公告、上传规范文档、行业报告 | 更新：2026-06-25

> ⚠️ 本文档是所有 Skill 开发的行为准则，每个 Skill 发布前必须逐条对照检查。

---

## 一、账号准入条件

| 条件 | 要求 | 当前状态 |
|------|------|----------|
| 粉丝数 | ≥ 1000 粉丝 | ❌ 0 粉 |
| 账号状态 | 正常，无违规记录 | ✅ 新账号 |
| 实名认证 | 已实名 | 待确认 |
| 内容质量 | 账号有活跃笔记 | ❌ 无内容 |

**应对策略**：先发布内容涨粉，同时走 ClawHub 和 GitHub 分发 Skill。

---

## 二、Skill 内容规范

### 2.1 鼓励的内容
- ✅ 实用的知识型 Skill（教程、指南、模板）
- ✅ 提高效率的工具型 Skill（自动化、计算器）
- ✅ 创意型 Skill（生成、设计、分析）
- ✅ 垂直领域的专业 Skill（餐饮、教育、金融）
- ✅ 原创内容，有明确的学习/使用价值

### 2.2 禁止的内容
- ❌ 涉及政治敏感话题
- ❌ 色情、低俗、擦边内容
- ❌ 赌博、彩票、虚拟货币交易
- ❌ 医疗建议、处方药推广
- ❌ 金融投资建议（非持牌）
- ❌ 侵犯他人知识产权
- ❌ 诱导用户跳转到违规平台
- ❌ 包含恶意代码或隐私窃取

### 2.3 格式要求
- 文件格式：`SKILL.md`（Markdown）
- 字数：无明确上限，建议 2000-10000 字
- 必须包含：名称、描述、使用方法、注意事项
- 支持图片：但必须原创或有授权

---

## 三、上传与审核

### 3.1 上传入口
1. 打开小红书 APP → 发现页
2. 搜索 `RED Skill` 进入频道
3. 点击「创建 Skill」按钮
4. 上传 SKILL.md 文件
5. 填写 Skill 信息（名称、描述、封面图）

### 3.2 审核标准
| 审核项 | 要求 |
|--------|------|
| 内容合规 | 不违反平台内容规范 |
| 格式正确 | SKILL.md 格式规范 |
| 功能可用 | AI 能正确理解并执行 |
| 描述匹配 | 描述与功能一致 |
| 封面规范 | 清晰、美观、无违规内容 |

### 3.3 审核周期
- 正常：1-3 个工作日
- 高峰：3-5 个工作日
- 被驳回：修改后重新提交

### 3.4 常见驳回原因
1. SKILL.md 格式不规范
2. 描述与实际功能不符
3. 包含违规内容
4. 封面图含有水印/广告
5. AI 无法正确理解 Prompt

---

## 四、安全与隐私

### 4.1 用户数据
- Skill 不应收集用户个人数据
- 如需数据，必须明确声明用途和存储方式
- 禁止将数据用于非 Skill 功能的用途

### 4.2 权限声明
- 涉及网络请求的 Skill 必须声明
- 涉及文件操作的 Skill 必须声明
- 涉及第三方 API 的 Skill 必须声明

### 4.3 免责声明
每个 Skill 建议包含：
```
免责声明：本 Skill 提供的建议仅供参考，不构成任何专业意见。
用户需自行判断和承担使用风险。
```

---

## 五、平台处罚机制

| 违规等级 | 行为 | 处罚 |
|----------|------|------|
| 轻度 | 描述不准确、格式问题 | 驳回 + 修改建议 |
| 中度 | 内容擦边、功能异常 | 驳回 + 警告 |
| 严重 | 违法内容、恶意行为 | 永久封禁账号 |

---

## 六、我们的合规承诺

1. **每个 Skill 发布前进行完整的自检**
2. **不确定的内容先问平台客服**
3. **第一时间响应平台审核意见**
4. **定期回顾更新平台规则变化**
5. **宁可不上架，不违规上架**
